# **Couverture du code modifié**

La **couverture du code modifié** vous permet de vous concentrer sur les tests du code modifié lors de cette modification.

## Dimensions


### **Couverture des lignes (fichier) ajoutées**

**Logique de calcul :**
1. git diff entre deux commits pour obtenir les fichiers modifiés => numéros de lignes modifiées (y compris les commentaires et les lignes vides)
2. Parcourir **branchMap**, **fnMap** et **statementMap** de **coverageMap** des fichiers modifiés pour vérifier si ils incluent les numéros de lignes modifiées
3. **Définition de "couvert"** : Les **branchMap**, **fnMap** et **statementMap** où se trouvent les numéros de lignes modifiées sont tous couverts

**Formule de calcul :**


```
Couverture des lignes (fichier) ajoutées = Nombre de lignes (fichier) couvertes / Nombre total de lignes (fichier) ajoutées
```
