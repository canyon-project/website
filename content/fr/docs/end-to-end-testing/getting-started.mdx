# Démarrage

Contrairement aux tests unitaires, les tests end-to-end ouvrent un navigateur et notre objectif est de collecter des données de couverture dans le navigateur. La complexité est donc relativement plus élevée. Voici le processus général.

- Exposer les variables globales du navigateur et écrire les données de couverture localement au bon moment
- Après la fin de l'automatisation UI, utiliser [canyon-uploader](/docs/ecosystem/canyon-uploader) pour lire et agréger les données de couverture du répertoire local `.canyon_output`, puis les rapporter.
- Associer l'ID du cas de test à l' [ID de rapport](/docs/core-concepts/report-id) pour garantir la traçabilité des cas de test.
