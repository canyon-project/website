# Configuration et Accès

Découvrez comment configurer et accéder à l'édition communautaire de Canyon.

## Configuration du Système

Pour utiliser toutes les fonctionnalités de Canyon, vous devez configurer les tables de données suivantes :

1. `CANYON_SERVER` : Adresse du serveur Canyon
2. `GITLAB_SERVER` : Adresse du serveur GitLab
3. `GITLAB_CLIENT_ID` : ID client GitLab
4. `GITLAB_CLIENT_SECRET` : Secret client GitLab

NOTE : L'URL de redirection est `<CANYON_SERVER>/oauth`, assurez-vous de la configurer correctement dans GitLab.

import { Callout } from 'nextra/components';

```sql copy
insert into public.sys_setting (id, key, value)
values  ('1', 'CANYON_SERVER', 'https://canyonjs.org'),
        ('2', 'GITLAB_SERVER', 'https://gitlab.com'),
        ('3', 'GITLAB_CLIENT_ID', 'xxx'),
        ('4', 'GITLAB_CLIENT_SECRET', 'xxx');
```

<Callout type="info" emoji="ℹ️">
  [Configurer GitLab en tant que fournisseur d'authentification OAuth 2.0](https://docs.gitlab.com/ee/integration/oauth_provider.html)
</Callout>

Actuellement, seul GitLab est pris en charge. D'autres fournisseurs Git seront pris en charge ultérieurement.

```sql copy
insert into public.git_provider (id, url, type, name, disabled, private_token)
values  ('gitlab', 'https://gitlab.com', 'gitlab', 'GitLab', false, 'xxx');
```

<Callout type="info" emoji="ℹ️">
  [Générer un GitLab Private Token](https://docs.gitlab.com/user/profile/personal_access_tokens/#create-a-personal-access-token)
</Callout>
