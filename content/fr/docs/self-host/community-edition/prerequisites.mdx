# Prérequis

Les conditions préalables à l'installation de Canyon sur votre propre infrastructure.

Canyon est une plateforme de développement API auto-hébergée, empaquetée sous forme de conteneurs Docker. Vous pouvez installer et exécuter Canyon sur n'importe quel système d'exploitation prenant en charge [Docker Engine](https://docs.docker.com/engine). Vous pouvez utiliser Canyon sur votre ordinateur local ou sur le fournisseur de cloud de votre choix.

## Exigences système

Canyon est conçu pour fonctionner efficacement dans des déploiements petits et grands. Les exigences minimales pour exécuter Canyon sont un système d'exploitation prenant en charge Docker, 4 cœurs CPU + 4 Go de RAM pour générer les images de construction, et au moins 1 cœur CPU + 2 Go de RAM pour héberger les fichiers de sortie générés.

### Installation de Node.js, npm, pnpm

#### Node.js + npm

Installez [`Node.js`](https://nodejs.org/en) (v18+) et [`npm`](https://www.npmjs.com) (v9+).

- [Guide d'installation Node.js + npm](https://nodejs.org/en/download)


### pnpm

Installez [`pnpm`](https://pnpm.io) (v9 recommandé).

- [Guide d'installation pnpm](https://pnpm.io/installation)

### Docker

Installez [`Docker`](https://www.docker.com) (v20+).

- [Guide d'installation Docker](https://docs.docker.com/engine/install)

### Base de données Postgres

Canyon utilise une base de données Postgres pour stocker les données de base (version 14.x ou supérieure requise). Vous pouvez utiliser n'importe quel fournisseur de base de données Postgres de votre choix - hébergé localement ou sur un fournisseur de cloud. Assurez-vous de disposer d'une URL de base de données Postgres valide au format suivant :

```
postgres://username:password@localhost:5432/dbname
```

#### Création des tables

```shell
pnpm run migrate
```

Ou utilisez directement le script [SQL](https://github.com/canyon-project/canyon/blob/main/packages/canyon-backend/prisma/create_table.sql) de création de tables.
