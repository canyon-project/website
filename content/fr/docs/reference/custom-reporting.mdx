# Rapport de Données Personnalisé

Dans certains scénarios, en particulier lorsque des pages H5 intégrées dans des applications mobiles doivent rapporter des données de couverture, il est nécessaire de mettre en œuvre une fonctionnalité de rapport de données de couverture personnalisée car il n'existe pas de plugin Chrome Canyon similaire à celui disponible sur les ordinateurs de bureau.

## Scénarios d'utilisation

- Pages H5 intégrées dans des applications mobiles
- Environnements où le plugin Chrome Canyon ne peut pas être utilisé
- Scénarios nécessitant une logique de rapport personnalisée
- Applications Web mobiles

## Solution de mise en œuvre

Voici une solution de base pour la mise en œuvre d'un rapport personnalisé :

```javascript copy
const coverageFirstValue = Object.values(window.__coverage__||{})[0];
if (coverageFirstValue) {
  fetch(coverageFirstValue.dsn,{
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${coverageFirstValue.reporter}`
    },
    body: JSON.stringify({
      coverage: window.__coverage__,
      ...coverageFirstValue
    })
  })
}
```
