# 소개

## 개요

Canyon(의미: "협곡", 발음: /ˈkænjən/)은 JavaScript 코드 커버리지 수집 플랫폼입니다. 개발자와 QA 엔지니어가 엔드투엔드 테스트 과정에서 테스트 케이스 코드 커버리지를 수집할 때 겪는 어려움을 해결합니다. 이는 세 가지 주요 부분으로 구성됩니다:

- 다양한 CI 제공업체에 적응하고 환경 변수를 감지하는 일련의 [플러그인](https://github.com/canyon-project/canyon/tree/main/plugins).

- 커버리지 데이터를 수집 처리하고 백엔드 API를 제공하는 [nodejs 서비스](https://github.com/canyon-project/canyon/tree/main/packages/canyon-backend).

- 커버리지 데이터를 표시하는 [react 프론트엔드 애플리케이션](https://github.com/canyon-project/canyon/tree/main/packages/canyon-platform).

![](https://cdn.jsdelivr.net/gh/canyon-project/assets/docs/static/docs/getting-started/introduction/home-screen.png)

## 왜 Canyon인가?

Canyon은 컴파일 단계에서 hit와 map 데이터를 분리함으로써 UI 자동화 테스트에서 생성된 대량의 커버리지 데이터를 효율적으로 처리할 수 있습니다.

또한, Canyon은 일반적인 CI 제공업체에 적응하여 빌드 단계에서 프로브 코드를 삽입하고, UI 자동화 테스트 단계에서 커버리지 데이터를 수집하고 보고할 수 있습니다.

이를 통해 개발 팀은 현재 UI 자동화 테스트의 __각 케이스__에 대한 커버리지 상황을 __정확하게__, __실시간으로__ 파악할 수 있어 코드 품질을 더 잘 평가하고 향상시킬 수 있습니다.

### 기능 특징

- [정확하고 효율적](/docs/core-concepts/separate-hit-and-map): hit와 map을 분리하여 컴파일 시 초기 커버리지 데이터를 생성하고, 정확하고 효율적으로 커버리지 데이터 수집
- [SourceMap](/docs/core-concepts/restore-source-code-coverage): 정확한 소스 코드 커버리지 복원
- [빌드 도구](/docs/installation/getting-started): next.js, vite, webpack 등 일반적인 빌드 도구의 커버리지 솔루션 제공
- [자동화 프레임워크](/docs/end-to-end-testing/getting-started): 일반적인 UI 자동화 프레임워크 통합 솔루션 제공
- [CI 제공업체](/docs/reference/provider): 일반적인 CI 제공업체(Github Actions, GitLab Runner)에 적응하고 CI 환경 변수 감지


### 자체 호스팅

커버리지 데이터와 테스트 데이터를 완전히 제어하고 싶다면, 자신의 인프라에서 Canyon을 [자체 호스팅](/docs/self-host/community-edition/prerequisites)할 수 있습니다.

## 아키텍처

다음 다이어그램은 Canyon 및 그 생태계의 일부 구성 요소 아키텍처를 보여줍니다:

1. Babel 플러그인이 CI/CD 파이프라인에서 코드 계측을 완료합니다.
2. 프로젝트가 테스트 환경에 배포되면 사용자는 UI 자동화 또는 수동 테스트를 선택하여 계측된 코드 프로브를 트리거할 수 있습니다.
3. 생성된 커버리지 데이터는 Canyon 서버에 보고됩니다.
4. Canyon 서버는 데이터를 소비하고 GitLab에서 제공하는 소스 코드 정보와 결합하여 커버리지 보고서를 생성합니다.
