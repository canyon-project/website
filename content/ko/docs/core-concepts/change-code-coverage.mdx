# **변경 코드 커버리지**

**변경 코드 커버리지**를 통해 이번 코드 변경에 대한 테스트 상황에 집중할 수 있습니다.

## 차원


### **추가된 (파일) 라인 커버리지**

**계산 로직:**
1. git diff로 두 커밋 간의 차이 파일 => 차이 라인 번호(주석, 공백 라인 포함 가능)
2. 변경된 파일의 **coverageMap**에서 **branchMap**, **fnMap**, **statementMap**의 위치가 차이 라인 번호를 포함하는지 확인
3. **커버됨**의 정의: 차이 라인 번호가 위치한 **branchMap**, **fnMap**, **statementMap**이 모두 커버됨

**계산 공식:**


```
추가된 (파일) 라인 커버리지 = 커버된 (파일) 라인 수 / 추가된 (파일) 라인 총 수
```
