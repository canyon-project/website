# 시작하기

단위 테스트와 달리, 엔드 투 엔드 테스트는 브라우저를 실행하며, 우리의 목표는 브라우저 내의 커버리지 데이터를 수집하는 것입니다. 따라서 복잡도가 상대적으로 높습니다. 다음은 대략적인 프로세스입니다.

- 브라우저의 전역 변수를 노출시키고, 적절한 시점에 커버리지 데이터를 로컬에 저장합니다.
- UI 자동화가 완료된 후 [canyon-uploader](/docs/ecosystem/canyon-uploader)를 사용하여 로컬 `.canyon_output` 디렉토리에 있는 커버리지 데이터를 읽어 집계하여 보고합니다.
- caseID를 [Report ID](/docs/core-concepts/report-id)와 연결하여 케이스를 추적할 수 있도록 합니다.
