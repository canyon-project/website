# 설정 및 액세스

Canyon 커뮤니티 에디션을 설정하고 액세스하는 방법을 알아보세요.

## 시스템 설정

Canyon의 모든 기능을 사용하려면 다음 데이터 테이블을 구성해야 합니다:

1. `CANYON_SERVER`: Canyon 서버 주소
2. `GITLAB_SERVER`: GitLab 서버 주소
3. `GITLAB_CLIENT_ID`: GitLab 클라이언트 ID
4. `GITLAB_CLIENT_SECRET`: GitLab 클라이언트 비밀 키

참고: 콜백 URL은 `<CANYON_SERVER>/oauth`입니다. GitLab에서 올바르게 구성되었는지 확인하세요.

import { Callout } from 'nextra/components';

```sql copy
insert into public.sys_setting (id, key, value)
values  ('1', 'CANYON_SERVER', 'https://canyonjs.org'),
        ('2', 'GITLAB_SERVER', 'https://gitlab.com'),
        ('3', 'GITLAB_CLIENT_ID', 'xxx'),
        ('4', 'GITLAB_CLIENT_SECRET', 'xxx');
```

<Callout type="info" emoji="ℹ️">
  [GitLab을 OAuth 2.0 인증 공급자로 구성](https://docs.gitlab.com/ee/integration/oauth_provider.html)
</Callout>

현재 GitLab만 지원되며, 나중에 더 많은 Git 공급자가 지원될 예정입니다.

```sql copy
insert into public.git_provider (id, url, type, name, disabled, private_token)
values  ('gitlab', 'https://gitlab.com', 'gitlab', 'GitLab', false, 'xxx');
```

<Callout type="info" emoji="ℹ️">
  [GitLab 개인 토큰 생성](https://docs.gitlab.com/user/profile/personal_access_tokens/#create-a-personal-access-token)
</Callout>
